# User Rules

1. **Design Alignment**: Always design according to the existing application's character and style.
2. **Scope Separation**: When working on design, do NOT modify functional code.
3. **Core Mechanics Protection**: Do NOT modify the game's core rules or mechanics.
4. **Modification Protocol**: If functional changes are necessary, you **MUST** first ask for permission in **Korean** explaining why the change is needed.
5. **Device Specific Design**: Treat iPhone and iPad designs as completely separate projects. Do NOT modify iPad design/code when working on iPhone, and vice-versa. Do not assume they share design principles.
6. do not use snackbar

## Design Identity: Sketch & Cartoon
Always adhere to the following visual principles:
- **Core Aesthetic**: Hand-drawn 'Sketch' style on a paper-like background.
- **Color Palette**: Off-white/Beige (#FAF7F0) base with high-contrast vibrant cartoon colors (Red, Blue, Yellow, Green).
- **Ink Outlines**: Everything MUST have a thick black ink outline (2-4px). Avoid soft shadows or gradients unless they fit the 'marker' look.
- **Offset Shadows**: Use hard, offset shadows to create a 'paper-cutout' or 'layered' feel.
- **Visual Impact**: Use large, bold, outlined damage numbers for tactical feedback.
- **Typography**: Bold, high-impact, and clear. Avoid thin or delicate fonts.
